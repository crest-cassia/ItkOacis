    class ItkOacis::ConductorRandom

    Conductor that manages to create new ParamSetStub for random-search. A policy to scatter ParamSets can be specified in conf parameter in new or DefaultConf constant defined in sub-classes. (See DefaultConf for the syntax of the specification.)

    Usage¶ ↑

    ## add path for "Conductor.rb" to $LOAD_PATH.
require 'ConductorRandom.rb' ;

class FooConductor < ItkOacis::ConductorRandom
  ## override DefaultConf.
  DefaultConf = {
    :simulatorName => "foo00",
    :hostName => "localhost",
    :scatterPolicy => { "x" => { :type => :uniform,
                                 :min => -1.0, :max => 1.0 },
                        "y" => { :type => :gaussian,
                                 :ave => 10.0, :std => 1.0 },
                        "z" => { :type => :list,
                                 :list => [0.0, 1.0, 2.0, 3.0] } }
  } ;

end

# create a FooConductor and run.
conductor = FooConductor.new() ;
conductor.run() ;

    Constants

    DefaultConf

    default values of conf in new method. It should be a Hash. It overrides Conductor::DefaultConf. See below for meaning of each key: (See also ItkOacis::Conductor::DefaultConf)

    • :scatterPolicy : define a policy to scatter parameter values. (default: {}) Detailed syntax of the specification is as follows:

      <Conf> ::= { ...
             :scatterPolicy => { <ParamName> => <RandPolicy>,
                                 <ParamName> => <RandPolicy>,
                                 ... },
             ... }
<ParamName> ::=  a string of the name of a parameter.
<RandPolicy> ::= { :type => :uniform, :min => min, :max => max }
               | { :type => :gaussian, :ave => ave, :std => std }
               | { :type => :value, :value => value }
               | { :type => :list, :list => [value, value, ...] }

    Attributes

    scatterPolicy[R]

    scatter policy definition. The name is specified in conf in new method by key :scatterPolicy.
