    class ItkOacis::ConductorGaProgressive

    Conductor that manages to ParamSet according to a progressive GA (Genetic Algorithm) way.

    Compared with ConductorGaSimple, ConductorGaProgressive alternate the next generation when a part of generation have done the calculation. In default, it use tournament4 () to alter the partial generation.

    In the tournament4 (), the trigger to start the next generation when 4 ParamSet are completed. The 4 ParamSet are sorted and generate the next 4 ParamSet and submit them. (See tournament4 () for more detail.)

    Usage¶ ↑

    class FooConductor < ItkOacis::ConductorGaProgressive
  ## default configulation for initialization.
  DefaultConf = {
    :simulatorName => "foo00",
    :hostName => "localhost",
    :scatterPolicy => { "x" => { :type => :uniform,
                                 :min => -1.0, :max => 1.0 },
                        "y" => { :type => :gaussian,
                                 :ave => 10.0, :std => 1.0 },
                        "z" => { :type => :list,
                                 :list => [0, 1, 2, 3] } },
    :population => 10,      
    :nofAlternation => 10,
    :mutateRange => { "x" => { :type => :uniform,
                               :min => -0.1, :max => 0.1 },
                      "y" => { :type => :gaussian,
                               :ave => 0.0, :std => 0.1 } },
    :logLevel => :info,
  } ;

  ## to compare two ParamSetStub.
  def scoreOf(_psStub)
    _r = _psStub.getResultTable(:average) ;
    _v = _r["u"] + _r["v"] + _r["w"] ; 

    return _v ;
  end

end 

# create a FooConductor and run.
conductor = FooConductor.new({:nofAlternation => 100}) ;
conductor.run() ;

    Constants

    DefaultConf

    default values of conf in new method. It should be a Hash. It overrides Conductor::DefaultConf. See below for meaning of each key: (See also ItkOacis::ConductorGaSimple::DefaultConf)

    • :tournamentBy : the procedure to partial tournament. If nil, use the default (tournament4 ()) method. If a Proc, call it with 4 ParamSetStub. (default: nil)

    See below for syntax of each key:

    <Conf> ::= { ...
             :tournamentBy => <MethodSpec>,
             :nofPlayer => 4,
             ... }
<MethodSpec> ::= nil | a Proc.

    Attributes

    nofPlayer[R]

    the number of tournament player. It is specified in conf in new method by key :nofPlayer.

    tournamentBy[R]

    the procedure to do tournament of ParamSetStub to create new generation. It is specified in conf in new method by key :tournamentBy.

    tournament4(_playerList) click to toggle source

    to alternate generation by tournament4. It takes playerList whose length is 4 (or more). It sorts the list and generates new 4 as follow.

    • a surviver who is first in the sorted list.

    • a mutant who is mutated from the surviver.

    • a crossover whose parents are the surviver and the second one in the list.

    • a random one who is generated in the same way of initial population.

    It can be overrided by expanded classes.

        # File ConductorGaProgressive.rb
240 def tournament4(_playerList)
241   # sort.
242   _playerList.sort!(&@compareBy) ;
243 
244   # survive
245   _surviver = _playerList.first() ;
246   @runningParamSetList.push(_surviver) ;
247 
248   # mutate
249   _mutantSeed = @mutateBy.call(_surviver) ;
250   _mutant = spawnParamSet(_mutantSeed) ;
251 
252   # crossOver
253   _second = _playerList[1] ;
254   _crossSeed = @crossOverBy.call(_surviver, _second) ;
255   _cross = spawnParamSet(_crossSeed) ;
256 
257   # rest, random.
258   _restList = fillRunningParamSetList() ;
259 
260   # setup pending and waiting.
261   @runningParamSetList.delete(_surviver) ;
262   _waitList = [_mutant, _cross] + _restList ;
263   _pendingInfo = { :wait => _waitList, 
264                    :done => [] } ;
265   @pendingPool[_surviver] = _pendingInfo ;
266   _waitList.each{|_wait|
267     @waitTable[_wait] = _surviver ;
268   }
269   
270 end
